# Contributing to prince2-agile-templates

Thank you for your interest in contributing. This library is built by practitioners for practitioners — if you have a template that has served you well on a real project, it belongs here.

---

## What we are looking for

- **New templates** — particularly for sectors not yet covered: defence, healthcare, government, construction, financial services
- **Improvements to existing templates** — additional fields, clearer definitions, better examples
- **Translations** — templates in other languages for international PM communities
- **Example completed templates** — anonymised real-world examples showing how to fill in the templates
- **Corrections** — typos, broken formatting, or unclear instructions

---

## How to contribute

1. **Fork the repo** — click Fork at the top of the page
2. **Create a branch** — `git checkout -b feat/your-template-name`
3. **Add your template** to the `templates/` folder
4. **Update the README** — add your template to the table with a one-line description
5. **Submit a pull request** — describe what the template is for and where it came from

---

## Template standards

Every template should follow this structure:

```markdown
# Template Title
# Usage: One sentence explaining when and how to use this template.
# MIT Licence — Copyright (c) 2026 Temitope Kadri

---

## [Section headings]

[Content]

---

*Template version 1.0 — prince2-agile-templates*
*github.com/TemitopeKadri/prince2-agile-templates*
```

**Rules:**
- Fields that must be completed are marked `[REQUIRED]` or use `[placeholder]` text
- Include definitions for any scoring, status, or categorisation fields
- Keep language plain and direct — no jargon without explanation
- If the template has a PRINCE2 Agile integration angle, include a section for it
- File naming: `kebab-case.md` e.g. `risk-register.md`

---

## What we are NOT looking for

- Templates that duplicate existing ones without meaningful improvement
- Templates for non-project-management purposes
- Proprietary or confidential templates from your employer
- Templates that require specific paid tools to use

---

## Reporting issues

If an existing template has an error, unclear instruction, or missing field, open an issue describing:
- Which template
- What the problem is
- What you would suggest instead

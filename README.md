# prince2-agile-templates

**MIT-licensed stage-gate templates, RAID schemas, and delivery governance frameworks for regulated project environments.**

A curated library of practical delivery templates built from real programme delivery across telecoms infrastructure, ERP implementations, data centre commissioning, and AI SaaS environments. Designed for practitioners running hybrid PRINCE2/Agile delivery in regulated or complex settings.

---

## What's in here

| Template | Format | What it's for |
|----------|--------|---------------|
| Stage Gate Review | Markdown | End-of-stage assessment and go/no-go decision |
| RAID Log | CSV + MD | Risks, Assumptions, Issues, Dependencies tracking |
| Definition of Done | Markdown | Agile DoD extensions for regulated delivery |
| Steering Committee Pack | Markdown | Weekly/monthly executive status report template |
| Sprint Review Template | Markdown | Sprint review format for hybrid PRINCE2/Agile teams |
| Change Control Log | CSV | Scope change tracking and impact assessment |
| Lessons Learned Register | CSV | Structured lessons capture for post-project review |
| Cutover Checklist | Markdown | Pre/during/post go-live verification checklist |
| Benefits Realisation Plan | Markdown | Benefits tracking from business case to closure |
| Stakeholder Register | CSV | Stakeholder mapping with influence/interest scoring |

---

## Who this is for

- Delivery managers running hybrid PRINCE2/Agile programmes
- Project managers in regulated industries (financial services, telecoms, infrastructure, healthcare)
- PMO teams building a reusable template library
- Anyone studying for PRINCE2, PRINCE2 Agile, or APM qualifications

---

## How to use

Clone or download the repo. Templates are in `/templates` organised by category. Each template includes:
- A completed example (in `examples/`)
- Usage notes at the top of the file
- Fields marked `[REQUIRED]` vs `[OPTIONAL]`

## T-48 Hours

- [ ] Final UAT sign-off confirmed and documented
- [ ] All critical and high defects resolved and retested
- [ ] Production environment provisioned and validated
- [ ] Data migration dry-run completed successfully
- [ ] Rollback plan documented and agreed
- [ ] Go/No-Go meeting scheduled and attendees confirmed
- [ ] Communications drafted and approved (go-live announcement, support contacts)
- [ ] Hypercare support rota confirmed and published
- [ ] Training completion confirmed for all in-scope users

---

## T-24 Hours

- [ ] Go/No-Go meeting held — decision: GO / NO-GO
- [ ] Final data backup taken and verified
- [ ] All stakeholders notified of confirmed go-live time
- [ ] Support team briefed and on standby
- [ ] Legacy system freeze communicated and scheduled
- [ ] Final access and permissions checks completed
- [ ] Out-of-hours contact list confirmed for all key roles

---

## T-4 Hours

- [ ] Legacy system frozen (if applicable)
- [ ] Final pre-cutover backup completed
- [ ] Infrastructure readiness confirmed (network, firewall, DNS)
- [ ] War room / cutover team assembled
- [ ] Communication channel open (Teams bridge, conference line)

---

## Go-Live (T-0)

- [ ] Cutover runbook execution begins
- [ ] Each step signed off in real time
- [ ] Rollback triggers monitored throughout
- [ ] Smoke test completed and passed
- [ ] Go-live confirmed to all stakeholders

---

## T+4 Hours (Immediate Post Go-Live)

- [ ] Initial system stability confirmed
- [ ] First wave of users able to log in and complete a core transaction
- [ ] No critical incidents reported
- [ ] Support queue monitored actively

---

## T+24 Hours

- [ ] Full regression check completed
- [ ] Performance baseline confirmed within SLA
- [ ] Support ticket volume reviewed
- [ ] First post go-live status update sent to stakeholders

---

## T+1 Week

- [ ] Hypercare review conducted
- [ ] Open issues triaged and assigned
- [ ] User feedback collected
- [ ] Adoption metrics first reading taken

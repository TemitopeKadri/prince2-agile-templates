Standard Definition of Done
A user story or task is DONE when all of the following are true:
Code and Development
[ ] Code reviewed by at least one peer
[ ] Unit tests written and passing
[ ] No critical or high severity bugs introduced
[ ] Code merged to main branch
Quality and Testing
[ ] Functional testing completed against acceptance criteria
[ ] Regression test passed (no existing functionality broken)
[ ] Performance tested where applicable
[ ] Accessibility checked where applicable
Documentation
[ ] Technical documentation updated
[ ] User-facing documentation updated or confirmed not required
[ ] RAID log updated if new risks, issues or dependencies identified
Governance (PRINCE2 Agile extensions)
[ ] Change log updated if scope was modified
[ ] Product Description acceptance criteria met and signed off by Product Owner
[ ] Lessons learned captured if applicable
[ ] Stage gate criteria reviewed if end of stage
Deployment
[ ] Successfully deployed to test environment
[ ] Smoke test passed in test environment
[ ] Release notes drafted
---
Definition of Done — Go-Live / Cutover
A release or go-live is DONE when all of the following are true:
[ ] All user stories in scope marked as Done per standard DoD above
[ ] UAT sign-off obtained from named business owner
[ ] Cutover runbook executed and all steps signed off
[ ] Go / No-Go gate passed with named decision authority
[ ] Hypercare support rota confirmed and communicated
[ ] Rollback plan confirmed as no longer required or standing down
[ ] BAU handover criteria met and signed off
[ ] Post-implementation review scheduled
---
Definition of Done — Stage Gate (PRINCE2)
A stage is DONE when all of the following are true:
[ ] All planned deliverables accepted by Project Board
[ ] Stage budget reconciled and variance explained
[ ] RAID log reviewed and updated
[ ] Lessons learned register updated
[ ] Next stage plan approved by Project Board
[ ] Stage gate review document signed off
